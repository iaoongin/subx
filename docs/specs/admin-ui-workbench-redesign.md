# 后台管理台 UI 重构 Spec

## 背景
现有后台首页把系统配置、分组管理、分组内操作、订阅列表操作堆叠在同一层，存在这些问题：

- 操作入口重复，用户难以判断“现在操作的是全局还是当前分组”。
- 页面缺少稳定上下文，新增订阅、绑定订阅、刷新缓存这些动作和当前分组的关系不够明确。
- 列表虽然能展示订阅，但缺少筛选、搜索和空状态引导，页面在“分组为空”或“搜索无结果”时可读性差。
- 视觉层级不清晰，导致高频操作路径不够直接。

本方案目标是把后台首页重构为“分组上下文驱动的订阅工作台”。

## 目标

- 把“全局动作”和“当前分组动作”拆开。
- 让用户始终知道当前操作所在的分组。
- 把订阅列表改造成可搜索、可筛选、可读性更强的工作区。
- 为空状态、异常状态和加载状态提供明确反馈。
- 不改动现有后端接口契约，仅重构前端信息架构和交互逻辑。

## 非目标

- 不引入新的前端框架。
- 不改动后端路由设计。
- 不做分组级配置覆盖。
- 不处理登录页样式重构。

## 信息架构

### 顶层结构

页面拆成 4 个层次：

1. Hero 区
2. 分组统计区
3. 左侧分组上下文侧栏
4. 右侧订阅工作区

### Hero 区

职责：

- 作为页面总入口，承载产品标题和页面定位说明。
- 承载全局级动作，而不是分组内动作。

文案要求：

- 主标题使用 `SubX 订阅工作台`
- 副标题沿用历史 slogan：`轻松管理您的订阅链接，支持实时添加、编辑和删除`

包含动作：

- `管理分组`
- `系统配置`
- `退出登录`

### 分组统计区

职责：

- 显示当前分组维度下的聚合统计。

指标：

- 订阅总数
- 已启用数量
- 已停用数量

### 左侧分组上下文侧栏

职责：

- 明确“当前分组”。
- 提供与当前分组强绑定的操作入口。

区块拆分：

#### 1. 当前分组卡片

显示内容：

- 当前分组名称
- 当前分组连接状态
- 当前分组的订阅访问地址
- 当前分组 token
- 当前分组订阅数量

允许操作：

- 切换分组
- 复制当前分组订阅地址
- 预览订阅
- 刷新缓存
- 刷新流量

#### 2. 分组内操作卡片

允许操作：

- 新建订阅
- 绑定已有订阅

规则：

- 当没有选中分组时，上述按钮全部禁用。
- 新建订阅默认自动绑定到当前分组。

### 右侧订阅工作区

职责：

- 承载当前分组下的订阅列表和筛选能力。

区块拆分：

#### 1. 列表头部

显示内容：

- 标题：`订阅列表`
- 说明文案：当前仅展示所选分组内的订阅
- 结果计数：`筛选结果 / 当前分组总数`

#### 2. 工具栏

包含：

- 搜索框
- 状态筛选：`全部 / 启用中 / 已停用`
- 类型筛选：`全部类型 / 订阅链接 / 节点列表`

下拉控件要求：

- 页面内下拉控件统一使用自定义下拉组件
- 不直接暴露浏览器原生 `select` 外观
- 分组选择、类型筛选、表单类型选择、预览格式选择都必须使用同一套视觉和交互逻辑

#### 3. 卡片列表

每张卡片展示：

- 名称
- 类型标签
- 状态标签
- 描述
- 链接区域
- 流量与到期信息
- 操作按钮

链接区域规则：

- 订阅链接卡片主行只显示 `host`
- 订阅链接卡片副行显示“点击复制完整链接”
- 节点列表卡片主行显示“节点列表 + 条数”
- 节点列表卡片副行显示“点击复制节点内容”
- hover 可通过原始标题信息查看完整链接
- 点击链接区域时复制完整原始内容，而不是复制缩略文本

操作按钮：

- 复制
- 编辑
- 启用或停用
- 删除
- 解绑

## 交互规则

### 分组切换

- 切换分组时，重新加载当前分组订阅列表。
- 切换分组时，重新计算顶部统计数字。
- 切换分组时，刷新左侧分组摘要。
- 切换分组后，不重置搜索和筛选控件的值，但筛选作用于新分组数据集。

### 新建订阅

- 必须先选中分组。
- 创建成功后自动调用分组绑定接口。
- 若创建成功但绑定失败，需要给出明确错误提示。
- 创建完成后刷新当前分组列表。

### 绑定已有订阅

- 只展示当前分组尚未绑定的订阅。
- 成功绑定后关闭弹窗并刷新当前分组列表。

### 解绑订阅

- 解绑只移除当前分组关系，不删除订阅实体。
- 解绑成功后刷新当前分组列表。

### 删除订阅

- 删除为实体级删除。
- 删除成功后刷新当前分组列表。

### 搜索与筛选

- 搜索范围：名称、描述、链接。
- 状态筛选和类型筛选与搜索共同生效。
- 若结果为空，显示“无匹配结果”的专用空状态。

## 状态规则

### 无分组

表现：

- 左侧显示“还没有分组”。
- 右侧显示引导空状态。
- 需要分组上下文的按钮全部禁用。

### 未选中分组

表现：

- 左侧提示先选择分组。
- 右侧显示“还没有选中分组”空状态。

### 分组为空

表现：

- 显示“这个分组还是空的”空状态。
- 提供“新建订阅”快捷入口。

### 筛选无结果

表现：

- 显示“没有匹配结果”空状态。
- 提供“清空筛选”操作。

### 加载中

表现：

- 页面级请求显示全局 loading overlay。
- 使用量刷新使用静默加载，不重复打断主视图。

### 错误

表现：

- 列表区域显示错误态文案。
- Toast 用于补充说明失败原因。

## 视觉方向

目标视觉语言：

- 非传统后台灰白风格，改为更强层次的工作台式布局。
- 左右分栏明确职责。
- 强化当前分组的存在感。
- 卡片式订阅列表优先可读性，其次才是信息压缩。

视觉原则：

- Hero 区承担品牌感和顶层导航感。
- 侧栏承担“上下文感”。
- 列表区承担“效率感”。
- 颜色上避免默认紫白后台风格，使用暖色主品牌 + 绿色辅助信息色。
- 订阅卡片中的链接信息优先保证扫读效率，沿用旧版“主看 host，不铺开整段 URL”的展示逻辑。
- 停用态必须明显区别于启用态，优先沿用旧版“灰底、去饱和、弱交互”的表现，而不是仅做轻微透明度下降。
- 下拉控件必须摆脱原生浏览器样式，保持统一的面板化视觉风格。

## 前端实现映射

### 页面结构

- `public/index.html`

### 样式系统

- `public/index.css`

### 状态与交互逻辑

- `public/index.js`

职责拆分：

- `SubscriptionManager`
  - 订阅列表加载
  - 搜索和筛选
  - 卡片渲染
  - 新增、编辑、删除、启停
- `GroupManager`
  - 分组加载
  - 当前分组上下文渲染
  - 分组切换
  - 绑定和解绑
- `ConfigManager`
  - 系统配置读取与保存
- `CustomSelect`
  - 自定义下拉渲染
  - 原生 `select` 值同步
  - 下拉展开、收起、选中与聚焦

## 验收标准

- 页面初始加载后，若存在分组，默认进入首个分组。
- 当前分组名称、token、订阅地址和订阅数量必须同步显示。
- 无分组、空分组、筛选无结果三类空状态必须可区分。
- 搜索和筛选组合必须正确作用于当前分组内订阅。
- 新建订阅后必须自动绑定当前分组，否则显示明确失败提示。
- 删除、解绑、启停、绑定后都必须刷新当前分组列表。
- 所有分组依赖型按钮在无分组或未选择分组时必须禁用。
- 配置弹窗、分组弹窗、绑定弹窗、确认弹窗必须正常打开和关闭。
- 订阅链接卡片的链接区域必须默认显示 host，而不是截断后的完整 URL。
- 停用订阅卡片必须一眼可辨，至少同时体现灰底、灰阶化和 hover 弱化，不能只靠轻微置灰。
- 所有下拉控件必须使用统一的自定义组件，不能回退为原生浏览器样式。

## 后续可迭代方向

- 增加排序能力，如最近更新、流量占用、到期时间。
- 在分组卡片中增加更多分组维度指标，如节点列表数量、异常订阅数量。
- 为订阅卡片增加更紧凑的“密集模式”。
- 将流量状态做成更明确的风险提示，例如即将耗尽、已过期、缓存数据。
